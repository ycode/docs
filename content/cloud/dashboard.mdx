# Dashboard

The dashboard is the home screen of Ycode Cloud. It displays the projects in the active workspace and provides quick access to project creation, the editor, and billing.

## Workspace Switcher

The dashboard header shows the active workspace. Click it to open the workspace switcher and move between every workspace you belong to — personal, agency, or guest. The switcher also exposes a **New workspace** action that opens the workspace creation dialog (personal is free; agency requires a subscription).

When you change workspaces, the project list, **New project** action, and any workspace-specific navigation re-scope to the new context. Your last active workspace is remembered, so returning to the dashboard from the editor lands you back in the same one.

Workspaces you only have project-level access to appear in the switcher with a **Guest** badge. Inside a guest workspace you see only the projects you were specifically invited to, and the **New project** action is hidden.

See [Workspaces](/docs/cloud/workspaces) for the full breakdown of personal, agency, and guest workspaces.

## Project Sorting

Use the sort dropdown next to **New project** to reorder the dashboard:

- **Last viewed by me** (default) — projects you've opened recently float to the top; projects you've never opened keep their creation order
- **Last edited** — sorted by the most recent change to the project record (settings, billing, transfers, admin actions)
- **Alphabetically** — case-insensitive A → Z by project name

Your choice is remembered across reloads. The dropdown is hidden when the workspace has no projects yet.

## Project Cards

Each project appears as a card showing:

- **Thumbnail** — A preview image of the project (or a placeholder)
- **Project name** — The name you gave the project
- **URL** — Your custom domain if configured, otherwise your Ycode subdomain (`yoursite.ycode.website`)
- **Role badge** — If you're a collaborator (not the owner), your role is shown (Admin or Editor)

Click a project card to open it in the Ycode editor.

## Creating a Project

Click **New project** on the dashboard to open the project creation dialog. You have three options:

### Start from Scratch

Enter a project name and click **Create**. You'll get a blank project ready for building.

### Use a Template

Browse available templates in the gallery. Hover over a template to see options:

- **Use template** — Creates a new project with the template's design and content pre-applied
- **Preview** — Opens a live preview of the template in a new tab

### Import from Legacy

Transfer an existing Ycode Legacy project. See [Migrate from Legacy](/docs/getting-started/migrate-from-legacy) for the full process.

## Project Actions

Click the **⋮** menu on a project card for additional options:

- **Billing details** — Opens the Stripe customer portal for the project's subscription (personal workspaces only; agency workspace projects are billed at the workspace level)
- **Move to workspace** — Transfer the project to another workspace you own. See [Transferring Projects](/docs/cloud/workspaces#transferring-projects) for the billing impact
- **Delete** — Permanently removes the project and all its data

<Callout type="warning" title="Deleting projects">
Only the project owner can delete a project. Projects with an active paid subscription must have their subscription cancelled before deletion. Legacy imported projects can be deleted at any time.
</Callout>

## Project URLs

Each project gets a subdomain on `ycode.website` by default (e.g., `mysite.ycode.website`). You can connect a custom domain in the project's deployment settings. When a custom domain is configured, it appears on the dashboard card instead of the subdomain.

## Theme

Click the Ycode logo in the top-left of the dashboard header to open the theme menu and pick **System**, **Light**, or **Dark**. The preference is stored per-browser and applies to every dashboard page — login, onboarding, account, workspace settings, and the project grid. Choosing **System** follows your operating system's appearance preference live.

The dashboard theme is independent from the builder theme — switching one does not affect the other.

## Help and Support

The Ask Ycode chat bubble sits in the bottom-right of every authenticated dashboard page. It carries your conversation history across browsers and devices, and the current page context is sent with each message so answers can reference the screen you're on. Click the bubble to open the panel or close it when you're done.
