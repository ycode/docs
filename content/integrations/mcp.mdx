# MCP Server

Ycode includes an embedded [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) server that lets AI coding tools build and edit your site. Connect tools like Cursor, Claude Desktop, ChatGPT, or Windsurf to your Ycode project and use natural language to create pages, design layouts, manage CMS content, and publish changes.

## Setup

<Steps>

### Generate an MCP URL

1. In the Ycode editor, go to **Integrations** in the header
2. Open the **MCP** section
3. Click **Generate MCP URL**
4. Copy the generated URL (format: `https://your-site.com/ycode/mcp/ymc_...`)

### Connect your AI tool

**Cursor** ŌĆö Go to Settings, find MCP, click "Add server", select SSE type, and paste your MCP URL.

**Claude Desktop** ŌĆö Go to Settings, find Connectors, click "Add custom connector", and paste your MCP URL.

**ChatGPT** ŌĆö Enable developer mode, create a connector with your MCP URL, then attach it in a new chat. See [ChatGPT](#chatgpt) for the full steps.

### Start building

Ask the AI to create pages, add elements, style them, manage CMS content, and publish. Changes appear in the editor in real time.

</Steps>

## ChatGPT

ChatGPT can use your Ycode project over MCP as a remote **app** (OpenAI's UI may still say **connector** in places). You need [developer mode](https://platform.openai.com/docs/guides/developer-mode) turned on for your account, and your workspace must allow it.

<Steps>

### Turn on developer mode

In ChatGPT, open **Settings** ŌåÆ **Apps & Connectors** ŌåÆ **Advanced settings** (at the bottom) and enable **Developer mode**.

### Register your MCP URL

1. Open **Settings** ŌåÆ **Connectors** ŌåÆ **Create** (or **Apps** ŌåÆ **Create**, depending on your ChatGPT version).
2. Set **Connector name** to something you will recognize (for example, your site or ŌĆ£YcodeŌĆØ).
3. Set **Description** to what the integration does (for example, that it edits your Ycode site). The model uses this text to decide when to use the tools.
4. Paste your Ycode **MCP URL** (the same HTTPS URL from **Integrations ŌåÆ MCP** in the editor) as the **Connector URL** / server URL.
5. Save. If the connection succeeds, you should see the tools your project exposes. If you change tools later, open the connector in Settings and use **Refresh** to reload metadata.

### Use it in a chat

1. Start a **new** conversation.
2. Click the **+** button next to the message box, then **More**.
3. Choose your Ycode connector so its tools are available in that thread.
4. Prompt in plain language (for example, ŌĆ£Add a hero section to the home pageŌĆØ). ChatGPT may show tool calls and ask you to **confirm** before write actions run; you can approve each time or remember approvals for the conversation, depending on your settings.

</Steps>

## What AI Agents Can Do

The MCP server is at version **1.0**, with full feature parity with the Ycode editor. Tools are organized by area:

### Pages

Create, update, duplicate, and delete pages. Manage page settings including SEO metadata, custom code injection, and password protection. Organize pages into folders. Mark pages as the homepage (`is_index`), as a dynamic page (`is_dynamic`), or as an error page (`401`, `404`, `500`). Bind dynamic pages to a CMS collection with a slug field and control how `next-item` / `previous-item` link keywords traverse the collection.

### Layers and Elements

Add any element type ŌĆö including tables (`table`, `thead`, `tbody`, `tr`, `td`, `th`). Update design properties (layout, typography, spacing, sizing, backgrounds, borders, effects, position, transforms, transitions), text content, images, links, videos, and background images.

Layer settings cover the full surface: sliders (pagination, easing, touch events, slides per group), CMS-bound lightboxes (multi-image fields, groups), maps (Mapbox / Google Maps with coordinates, zoom, marker color, navigation controls), CMS-bound select/radio/checkbox options, plus per-layer `hidden`, `filter_on_change`, and placeholder flags.

Links support URLs, internal pages, email, phone, anchors (`anchor_layer_id`), specific dynamic items (`collection_item_id`), assets, plus `rel` and `download` attributes.

Move, reorder, and delete layers.

### Layouts

Insert pre-built layout templates ŌĆö hero sections, headers, feature grids, footers, and more. Layouts provide a quick starting point that the AI can then customize.

### CMS

Create and manage collections, fields, items, and item values. Field types include text, rich text, number, boolean, date, asset (image / audio / video / document, with optional `multiple`), reference, `option` (with custom choices), and `count` (computed from a referenced collection).

Add per-field metadata (`default`, `fillable`, `hidden`, `is_computed`), set a default `sorting` on each collection (`field`, `direction`), reorder fields with `reorder_collection_fields`, and reposition manually-sorted items with `set_collection_item_order`.

### Components

Create reusable components with variables, edit component layer trees, and manage the component library. Variables now include `rich_text` and `variant` types, plus `placeholder` and `default_value` fallbacks.

Variants have full CRUD: `list_component_variants`, `create_component_variant` (optionally cloning an existing variant), `update_component_variant`, and `delete_component_variant`. `update_component_layers` accepts an optional `variant_id` so edits target a specific variant without clobbering the others.

### Animations and Interactions

Add animations from 18 curated presets ŌĆö reveal, hover, click, parallax, stagger, and loop families ŌĆö in a single tool call:

- `add_animation` ŌĆö apply a preset to one or more `targets`
- `list_layer_animations`, `remove_layer_animation`, `clear_layer_animations`
- `set_layer_interactions` ŌĆö raw escape hatch for custom GSAP timelines

A dedicated **Animation Presets Reference** resource documents every preset, trigger semantics, and the raw interaction shape.

### Site Settings

Read and update project settings (site name, description, custom CSS, and more). Manage URL redirects with `list_redirects`, `add_redirect`, `update_redirect`, and `delete_redirect` ŌĆö including regex patterns and `301` / `302` types.

### Forms

View form submissions, update statuses, and manage form data. `update_form_settings` configures success behaviour (`message` or `redirect` to a URL) and email notifications (`to`, `subject`).

### Localization

Manage locales and translations for multilingual sites. `set_rich_text_translation` accepts structured rich-text blocks and writes a properly-formed translation ŌĆö no need to hand-assemble the underlying document JSON.

### Styles

Create, apply, update, and delete reusable layer styles for consistent design across the site.

### Assets

Upload images and files (from URL or base64), list and organize assets in folders.

### Color Variables

Create, update, delete, and reorder global color variables.

### Fonts

List installed fonts, search Google Fonts, and add new fonts to the project.

### Export

`export_layer_html` renders any layer subtree to a self-contained HTML string with Tailwind classes preserved ŌĆö useful for handing a finished section off to another tool or saving it as a snippet.

### Publishing

Check for unpublished changes and publish all drafts to make the site live.

## MCP Resources

The MCP server also provides read-only resources that give AI agents context about your project:

| Resource | Description |
|----------|-------------|
| **Elements Reference** | Available element types (including tables) and nesting rules |
| **Design Reference** | All design properties and valid values, across every category (including transforms and transitions) |
| **Animation Presets Reference** | The full preset catalog with trigger semantics, tween value formats, and a worked raw-interaction example |
| **Prompts Reference** | Example prompts and recommended workflows |
| **Site Pages** | Current pages and folder structure |
| **Site Collections** | CMS schema (collections, fields) |
| **Site Design Tokens** | Color variables, fonts, and locales |

## Managing Tokens

Each MCP URL contains a unique token. You can create multiple tokens (e.g., one per team member or AI tool) and revoke them individually from the Integrations page. Tokens authenticate all MCP requests ŌĆö treat them like API keys and do not share them publicly.
