# Design Controls

The Design tab in the right sidebar provides breakpoint-aware styling controls. All properties apply to the selected element and can vary by viewport size.

## Layout

Control how the element arranges its children:

- **display** — block, flex, grid, inline, inline-block, none
- **flexDirection** — row, column, row-reverse, column-reverse
- **flexWrap** — nowrap, wrap, wrap-reverse
- **justifyContent** — start, center, end, space-between, space-around, space-evenly
- **alignItems** — start, center, end, stretch, baseline
- **gap** — Spacing between flex or grid children
- **gridTemplateColumns** / **gridTemplateRows** — Grid column and row definitions
- **divideX** / **divideY** — Dividers between children

### Self Layout

When the selected layer is a **child** of a flex or grid container, a **Self layout** section appears with an `alignSelf` control that overrides the parent's `alignItems` for this one child. Options are Auto (no override), Start, Center, End, and Stretch — useful for un-stretching a single badge, pill, or button in an otherwise stretched flex column.

The icons rotate to match the parent's axis, and the parent type (e.g. Flex row) is shown in muted text so you can see what you're overriding. Self layout is available on text layers too, even though they don't show the main Layout panel.

## Typography

Control text appearance:

- **fontFamily** — Font family from your installed fonts (see [Fonts](/docs/editor/fonts) for adding Google Fonts and custom fonts)
- **fontWeight** — Normal, bold, or numeric weight (available weights depend on the selected font)
- **fontSize** — Text size
- **textAlign** — left, center, right, justify
- **letterSpacing** — Character spacing
- **lineHeight** — Line height
- **color** — Text color (solid, linear gradient, radial gradient, or image background clipped to text)
- **textTransform** — Add via the "+" menu: uppercase (default), lowercase, capitalize, or normal-case. Remove with the X button to clear the transform
- **textDecoration** — Add via the "+" menu: underline with configurable color, thickness, and offset
- **lineClamp** — Add via the "+" menu: limit text to a number of visible lines with overflow hidden (e.g., clamp to 2 lines for card excerpts)

For text layers, the color picker includes an **Image** tab where you can set a background image that clips to the text shape. Choose an image from the file manager, enter a custom URL, or bind to a CMS field. Configure size, position, and repeat settings for the image.

## Spacing

Set margin and padding using the visual box-model editor. The diagram shows margin (outer ring) and padding (inner ring), with inline inputs for each side.

- **margin** — All sides or individual top, right, bottom, left
- **padding** — All sides or individual sides
- **Shift + drag** — Update all sides at once
- **Alt + drag** — Update opposite sides together (top/bottom or left/right)

Values support px, rem, em, and percentage units. Use the **center block** button in the Spacing section header to set left and right margins to `auto`, horizontally centering the element within its parent.

## Sizing

Control element dimensions:

- **width** / **height** — Explicit dimensions (px, %, auto)
- **minWidth** / **minHeight** — Minimum constraints
- **maxWidth** / **maxHeight** — Maximum constraints
- **aspectRatio** — Add via the "+" menu with presets (Square, Video 16:9, 4:3, 3:4) or a custom ratio
- **objectFit** — cover, contain, fill, none (for images and videos)
- **objectPosition** — Pick a focal point for cropped images. The crosshair button next to Object fit opens a 3×3 grid; click any cell to align the image to that corner, edge, or center so the important part of the frame stays visible when the layer is cropped. Per-breakpoint overrides are supported.
- **overflow** — visible, hidden, scroll, auto, or **ellipsis**. Ellipsis truncates a single line of text with a trailing `…` (maps to Tailwind's `truncate`). Inside a flex container the text element may also need `min-w-0` to shrink and trigger the ellipsis.
- **gridColumnSpan** / **gridRowSpan** — Grid cell spanning

## Borders

Add and customize borders:

- **borderWidth** — All sides or individual sides
- **borderStyle** — solid, dashed, dotted, none
- **borderColor** — Border color
- **borderRadius** — Corner radius (all or per corner)

**Outline** — Add via the "+" menu in the Borders section. Configure outline width, color, and offset. Outlines render outside the border and do not affect layout, making them useful for focus indicators and decorative effects.

**Dividers** — Add via the "+" menu in the Borders section. Dividers create lines between child elements. Configure direction (vertical or horizontal), style (solid, dashed, dotted, double), and color. Remove dividers with the X button.

## Backgrounds

Set element backgrounds:

- **backgroundColor** — Solid color
- **backgroundImage** — URL or gradient
- **backgroundSize** — cover, contain, auto, or explicit dimensions
- **backgroundPosition** — top, center, bottom, or coordinates
- **backgroundRepeat** — repeat, no-repeat, repeat-x, repeat-y

## Effects

Effects are addable and removable. Use the "+" menu in the Effects section header to add individual effects, and the X button to remove them.

- **opacity** — 0 to 1
- **boxShadow** — Add via "+" menu; configure shadow offset, blur, spread, and color
- **blur** — Add via "+" menu; blur filter on the element
- **backdropBlur** — Add via "+" menu; blur behind the element
- **filter** — CSS filter values

## Transform

Transform properties are addable via the "+" menu. Use them to visually adjust elements without changing layout flow.

- **Scale** — Resize the element (with slider)
- **Rotate** — Rotate in degrees
- **Move** — Translate horizontally (X) and vertically (Y)
- **Skew** — Skew horizontally (X) and vertically (Y)
- **Transform origin** — Set the anchor point for transforms (center, corners, edges)

## Transition

Animate property changes with CSS transitions:

- **Property** — Which properties to animate: None, All, Default, Colors, Opacity, Shadow, or Transform
- **Duration** — Transition speed in milliseconds
- **Easing** — Add via "+" menu: Linear, Ease In, Ease Out, or Ease In Out
- **Delay** — Add via "+" menu: delay before the transition starts (ms)

## Position

Control positioning and stacking:

- **position** — static, relative, absolute, fixed, sticky
- **top** / **right** / **bottom** / **left** — Offset values
- **zIndex** — Stacking order

## UI States

Most design controls accept different values per **UI state** so the same element can look one way at rest, another on hover, focus, or while active. Open the **State selector** at the top of the Design panel and switch to the state you want to style — the rest of the panel keeps the same controls, but every change you make now writes to that state only.

| State | Activates when |
|-------|----------------|
| **Default** | Always (the base look) |
| **Hover** | The pointer is over the element |
| **Focus** | The element has keyboard or programmatic focus (inputs, buttons, links) |
| **Active** | The element is being pressed |
| **Disabled** | The element has the `disabled` attribute |
| **Current** | The element is a link or nav link that points to the page currently being viewed (`aria-current="page"`) — used for highlighting the active page in a navigation menu, the current pagination item, or the active slider bullet |

States stack with breakpoints, so a hover style set on desktop inherits down to tablet and mobile unless you override them per breakpoint. The **Current** state is available on any element that has a link configured, not just dedicated nav-link elements.

## Layer Styles

Layer Styles are reusable saved style sets. You apply them to elements to keep design consistent across the site. A layer can hold an **ordered stack** of Layer Styles — useful for building base + modifier patterns (`button` + `button-primary`, `card` + `card-featured`) and for high-fidelity Webflow paste, where combo classes import as stacked styles.

### The Stack Model

The Layer Styles panel shows the stack as an ordered row of chips, low priority on the left and high priority on the right. Each chip represents one applied style. Later chips win when properties conflict, exactly like CSS class order.

- **Active chip** — click a chip to make it active. The Design panel reflects that chip's values, and edits go to that style (or to a per-chip override).
- **Drag to reorder** — drag chips to change cascade order. Reordering re-flattens the stack into the rendered class string immediately.
- **Per-chip overrides** — when you tweak a property on a chip that's used elsewhere, Ycode marks it as a **Customized** override on this layer only, leaving the original style untouched.

### Working with Styles

- **Create** — style an element, then save the current styles as a Layer Style from the context menu or Design panel. Give it a name. Names don't need to be unique across the site — two unrelated styles can both be called `Card` if they belong to different sections.
- **Apply** — pick an existing Layer Style from the panel to stack it onto the active layer. Apply multiple styles to build combinations.
- **Customize / Update / Reset** — per chip: **Customize** turns the chip's current values into per-layer overrides, **Update** writes the chip's values back into the underlying Layer Style (affecting every layer that uses it), and **Reset** discards per-chip overrides and returns the chip to its saved values.
- **New** — forks the active chip in place: creates a new Layer Style from the current values and swaps the chip over to it, so you can branch a variant without breaking the original.
- **Detach** — unlinks the active chip from its Layer Style while keeping its visual properties as a local override on the layer. The other chips in the stack are unaffected.
- **Edit** — edit a Layer Style to update every layer that uses it. Changes propagate to every instance and every stack the style appears in.

<Callout type="info" title="Breakpoint-aware">
Every design control is breakpoint-aware. Set values for desktop (base), tablet (max-lg), and mobile (max-md). Smaller breakpoints inherit from larger ones unless you override.
</Callout>
